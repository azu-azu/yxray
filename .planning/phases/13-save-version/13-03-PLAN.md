**ChangesPanel.tsx**

Props interface:
```typescript
interface ChangesPanelProps {
  projectId: string
  projectPath: string
  hasAnyCommits: boolean
  onSaved: (save: LastSave) => void
  onDiscarded: () => void
}
```

State: `useState` for:
- `changedFiles: string[]` — loaded from GET /api/watch/status or passed in; populate via `useEffect` calling `GET /api/watch/status` on mount
- `checkedFiles: string[]` — initialized to all changedFiles (pre-checked per user decision)
- `commitMessage: string` — empty string initial
- `isSaving: boolean`
- `confirmDiscard: boolean` — drives AlertDialog open state

Behavior (per CONTEXT.md locked decisions):
- On mount: fetch `GET /api/watch/status?project_id={projectId}&folder={projectPath}` — use `changed_count`, `has_any_commits`, `total_workflows`; then fetch `GET /api/watch/status` again or use `git_changed_workflows` via a new endpoint — actually, fetch changed files from watch status isn't enough. Instead: add a second fetch to `GET /api/save/files?folder={projectPath}` ... but wait, that endpoint doesn't exist yet. Better: accept `changedFiles: string[]` as a prop from AppShell which fetches them. Update props to include `changedFiles: string[]`.
- Revised props: `{ projectId, projectPath, changedFiles, hasAnyCommits, totalWorkflows, onSaved, onDiscarded }`
- checkedFiles initialized to all changedFiles in useState initializer
- File list: render each file as `<div className="flex items-center gap-2"><Checkbox checked={...} onCheckedChange={...} /><span>{Path.basename}</span></div>` — basename via `file.split('/').pop() ?? file`
- When `!hasAnyCommits`: render amber callout card above file list with copy "First version save — This will save all {totalWorkflows} existing workflows in this folder as your starting point." Truncate file list to first 5 + "...and N more" when changedFiles.length > 5
- Commit message Textarea: placeholder changes based on hasAnyCommits — "e.g. Initial version of project workflows" when first save, "What changed? e.g. Updated filter logic" otherwise
- Save Version button: disabled when checkedFiles is empty OR isSaving is true. Empty message is allowed — defaults to "Save" server-side.
- On Save Version click: POST /api/save/commit with {project_id: projectId, folder: projectPath, files: checkedFiles, message: commitMessage}. On success: call `onSaved({ message: commitMessage || 'Save', fileCount: checkedFiles.length, savedAt: new Date() })`
- Discard button: opens AlertDialog. AlertDialog copy: "Discard changes to these {checkedFiles.length} workflow(s)? They'll be moved to .acd-backup — you can recover them from there." Buttons: [Cancel] [Discard]. On confirm: POST /api/save/discard with {project_id: projectId, folder: projectPath, files: checkedFiles}. On success: call `onDiscarded()`
