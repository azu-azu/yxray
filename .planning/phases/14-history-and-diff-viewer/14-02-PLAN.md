    Add two functions to the END of app/services/git_ops.py, after git_discard_files.

    Function 1 — git_log(folder: str) -> list[dict]:
    - Two-pass approach: first call gets commit headers with unit-separator format; second call per SHA gets changed files via git diff-tree
    - Step 1: subprocess.run(["git", "-C", folder, "log", "--pretty=format:%H\x1f%s\x1f%an\x1f%aI"], capture_output=True, text=True)
    - If returncode != 0 or stdout empty: return []
    - Parse: split each line on "\x1f" (3 splits max) → sha, message, author, timestamp
    - Step 2 per SHA: check has_parent via `git rev-parse --verify {sha}~1` (returncode == 0 means True)
    - Step 3 per SHA: get changed files via `git diff-tree --no-commit-id -r --name-only {sha}` → filter by WORKFLOW_SUFFIXES
    - Return list of dicts with keys: sha, message, author, timestamp, files_changed, has_parent

    Function 2 — git_show_file(folder: str, sha: str, filepath: str) -> bytes:
    - subprocess.run(["git", "-C", folder, "show", f"{sha}:{filepath}"], capture_output=True) — NOTE: no text=True, returns raw bytes
    - If returncode != 0: raise FileNotFoundError(f"{filepath} not found at {sha}")
    - Return result.stdout (bytes)

    Anti-pattern: do NOT use git log --name-only single-pass (blank-line parsing is fragile — see RESEARCH.md Pitfall 1).

    Helper function _run_diff(old_bytes, new_bytes, filename) -> str:
    - Use tempfile.mkstemp(suffix=".yxmd") pattern (Windows-safe — see RESEARCH.md Pitfall 3)
    - Write old_bytes to fd_a, close fd_a; write new_bytes to fd_b, close fd_b
    - Call pipeline_run(DiffRequest(path_a=pathlib.Path(path_a), path_b=pathlib.Path(path_b)))
    - Call HTMLRenderer().render(response.result, file_a=f"{filename} (previous)", file_b=f"{filename} (this version)")
    - In finally block: os.unlink(path_a), os.unlink(path_b)

    Endpoint 2 — get_diff:
    ```python
    @router.get("/{sha}/diff")
    def get_diff(sha: str, folder: str = Query(...), file: str = Query(...), compare_to: str | None = Query(None)):
        parent_sha = compare_to if compare_to else f"{sha}~1"
        has_parent = subprocess.run(
            ["git", "-C", folder, "rev-parse", "--verify", parent_sha],
            capture_output=True, text=True
        ).returncode == 0
        if not has_parent:
            return JSONResponse({"is_first_commit": True})
        try:
            old_bytes = git_ops.git_show_file(folder, parent_sha, file)
            new_bytes = git_ops.git_show_file(folder, sha, file)
        except FileNotFoundError as exc:
            raise HTTPException(status_code=404, detail=str(exc)) from exc
        html = _run_diff(old_bytes, new_bytes, file)
        return HTMLResponse(content=html)
    ```

    Note on route order: FastAPI matches routes in declaration order. The route "/{sha}/diff" will match any path segment for {sha} including a project_id string. Ensure "/{project_id}" route is declared FIRST and "/{sha}/diff" is declared second. FastAPI will match the longer path /abc123/diff to the second route correctly because it has a literal "/diff" suffix.
