Rewrite all three tests in tests/test_port_probe.py to use unittest.mock so they never bind to real ports.
The current tests bind to port 7433 on the real network stack, causing failures when the app is running
on that port (developer machine) or when CI has anything occupying that port.

Rewrite strategy (per RESEARCH.md recommendation):

**test_find_available_port_returns_7433:** Use `unittest.mock.patch('socket.socket')` to replace the
socket constructor with a factory returning a MagicMock whose `.bind()` always succeeds. The function
under test should see port 7433 as free and return it. Assert `port == 7433` and that the returned
object is the mock socket instance. Do NOT check `isinstance(sock, socket.socket)` — it will be a
MagicMock.

```python
from unittest.mock import MagicMock, patch

def test_find_available_port_returns_7433():
    """find_available_port() returns 7433 when it is free."""
    mock_sock = MagicMock()
    mock_sock.__enter__ = lambda s: s
    mock_sock.__exit__ = MagicMock(return_value=False)
    with patch('socket.socket', return_value=mock_sock):
        from app.main import find_available_port
        port, sock = find_available_port(start=7433, count=11)
    assert port == 7433
    assert sock is mock_sock
```

**test_find_available_port_skips_occupied:** Use real socket binding but on a high ephemeral port
range (19900–19910) instead of 7433–7443. Pre-bind port 19900. Assert function returns 19901 (the next
free port). This avoids conflicts with the running app on port 7433.

```python
def test_find_available_port_skips_occupied():
    """When start port is bound by another socket, returns next free port."""
    blocker = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    blocker.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    blocker.bind(("127.0.0.1", 19900))
    try:
        port, sock = find_available_port(start=19900, count=11)
        try:
            assert port == 19901
        finally:
            sock.close()
    finally:
        blocker.close()
```

**test_find_available_port_raises_when_all_full:** Same ephemeral port strategy — bind all of
19900–19910 (11 ports) then assert OSError. Do not use ports 7433–7443.

```python
def test_find_available_port_raises_when_all_full():
    """When all ports in range are bound, raises OSError."""
    blockers: list[socket.socket] = []
    try:
        for p in range(19900, 19911):
            s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
            s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
            s.bind(("127.0.0.1", p))
            blockers.append(s)
        with pytest.raises(OSError):
            find_available_port(start=19900, count=11)
    finally:
        for s in blockers:
            s.close()
```

Keep the module-level imports: `from __future__ import annotations`, `import socket`, `import pytest`,
`from unittest.mock import MagicMock, patch`, `from app.main import find_available_port`.
