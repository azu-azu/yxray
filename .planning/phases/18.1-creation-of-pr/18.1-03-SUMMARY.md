## Accomplishments

- Added `humanizeBranchName()` helper: strips `experiment/YYYY-MM-DD-` prefix, replaces hyphens with spaces, capitalizes first letter
- Extended `RemoteStatus` interface with `gitlab_repo_url`; updated `fetchStatus` to parallel-fetch both provider statuses
- Added PR state variables (`githubPRState`, `gitlabPRState`, `githubPRUrl`, `gitlabPRUrl`, `showGithubPRForm`, `showGitlabPRForm`, `prTitle`, `prDescription`)
- Added `fetchPRStatus(provider)` — calls `GET /api/remote/pr/status` on mount when `isExperiment`, sets `done` state if PR already exists
- Added `handleCreatePR(provider)` — calls `POST /api/remote/pr/create`, transitions to `done` with URL or `error`
- Added `renderPRSection(provider)` with three states: success link, inline form, and "Open pull request/merge request" button
- PR section gated: only visible when `isExperiment && repoUrl` (provider-specific); hidden on main branch or when no repo URL
- PR state resets (via separate useEffect) when activeProject or currentBranch changes
- Frontend builds with zero TypeScript errors; `npm run build` exits 0

## Decisions Made

- `fetchStatus` extended to parallel-fetch github and gitlab statuses so `remoteStatus.repo_url` (GitHub) and `remoteStatus.gitlab_repo_url` (GitLab) are both available — each tab's PR section gates on its own provider's URL
- `renderPRSection(provider)` as a single helper drives both tabs — all conditional labels, state refs, and visibility logic derived from the `provider` parameter; eliminates duplication
- `prTitle` and `prDescription` are shared state — the GitHub and GitLab tabs are mutually exclusive tab views so only one form is ever visible at a time; no per-provider form state needed
- Textarea imported from `@/components/ui/textarea` — consistent with existing shadcn component usage in the project
