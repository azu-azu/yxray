# Phase 18.1 Plan 02: PR/MR Router Endpoints Summary

**POST /api/remote/pr/create and GET /api/remote/pr/status endpoints added to remote router via TDD — delegates to Plan 01 service functions**

## Accomplishments

- Added `PRCreateRequest` model (project_id, folder, provider, title, description, branch)
- Added `POST /api/remote/pr/create` endpoint — delegates to `github_api.create_pull_request` / `gitlab_api.create_merge_request`; returns `{"pr_url": url}` on success, `{"success": False, "error": ...}` on any exception
- Added `GET /api/remote/pr/status` endpoint — returns `{"pr_exists": bool, "pr_url": str|None}`; gracefully returns `pr_exists: false` on missing config or any error
- Added `remote_auth.get_token(provider)` convenience dispatcher to `remote_auth.py`
- Updated `config_store.get_remote_repo` to accept an optional `provider` parameter — returns URL string when provider given (for new endpoints), dict otherwise (backward compatible)
- 4 new endpoint tests added to `tests/test_pr.py`; all 13 tests in `test_pr.py` pass; full suite 234 pass (3 pre-existing port_probe failures unchanged)

## Files Created/Modified

- `tests/test_pr.py` — 4 endpoint tests appended (test_post_pr_create_github, test_post_pr_create_error, test_get_pr_status_exists_github, test_get_pr_status_none_github)
- `app/routers/remote.py` — PRCreateRequest model, POST /api/remote/pr/create, GET /api/remote/pr/status endpoints added
- `app/services/remote_auth.py` — get_token(provider) dispatcher added
- `app/services/config_store.py` — get_remote_repo gains optional provider param

## Decisions Made

- `remote_auth.get_token(provider)` added as a convenience dispatcher so the router receives `"github"` / `"gitlab"` as a string and makes one call — avoids an if/else for token retrieval in every endpoint
- `config_store.get_remote_repo` updated with optional `provider` param: when provider given, returns the URL string directly (matching the plan's interface spec); existing callers with no provider arg still receive the full dict (backward compatible)
- `branch: str` is a required field on `PRCreateRequest` — the backend has no branch state so the frontend must always supply it; this is an explicit design decision noted in the plan

- **Fix:** Added `get_token(provider)` to `remote_auth.py` as a thin dispatcher
- **Files modified:** `app/services/remote_auth.py`

**2. [Rule 2 - Missing] Updated config_store.get_remote_repo with optional provider param**
- **Found during:** Task 1 GREEN
- **Issue:** Plan's interface spec described `config_store.get_remote_repo(project_id, provider) -> str | None`, but real function only accepted `project_id` and returned a dict. Tests mocked it returning a string directly.
- **Fix:** Added optional `provider` parameter; when given, returns `{provider}_url` value from the dict (string or None); when not given, returns full dict as before (backward compatible)
- **Files modified:** `app/services/config_store.py`
- **Commit:** `3743e88`
