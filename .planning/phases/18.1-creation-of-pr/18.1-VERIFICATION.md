## Design Deviation (Intentional)

The original plan placed PR creation UI in `RemotePanel.tsx` (inside the GitHub/GitLab tabs). During execution, this was changed at user direction: the "Open PR" button and inline form were moved to `HistoryPanel.tsx`, alongside the Push/Pull buttons. This is a better UX — PR creation is an action in the natural push → create PR workflow, not a config panel concern.

**All PR-UI requirements are met in `HistoryPanel.tsx` instead of `RemotePanel.tsx`.**
