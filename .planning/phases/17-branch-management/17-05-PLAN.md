    3. Chip should have neutral/muted style (not amber) on the main branch

    **Test 2 — Create experiment copy (BRANCH-01):**
    1. Click the branch chip
    2. Confirm: popover opens listing current branches with a checkmark on main
    3. Click `+ New experiment`
    4. Confirm: inline text field appears with placeholder "e.g. price-calc-test"
    5. Type a description (e.g. "test feature")
    6. Confirm: live preview shows "Will be: experiment/YYYY-MM-DD-test-feature" below the field

    **Test 3 — Switch between branches (BRANCH-02):**
    1. Click the branch chip again
    2. Confirm: both `main` and the experiment branch are listed
    3. Click `main` to switch back
    4. Confirm: chip returns to neutral `[⎇ main ▾]` style
    5. Click the chip again and switch back to the experiment branch

    3. Click the branch chip

    1. Switch to main branch first
    2. Click the branch chip
    3. On the experiment row, click the trash icon

    5. Click Delete
    6. Confirm: experiment branch removed from the list

    **Test 6 — History filtered to branch:**
    1. Create a new experiment copy
    2. Save a version while on the experiment branch
    3. Switch to History view
    4. Confirm: History shows only the commits made on this branch (main commits not visible)
    5. Switch back to main via the chip
    6. Confirm: History returns to showing main branch commits

    **Test 7 — DiffViewer compare toggle (if on experiment branch with commits):**
    1. While on an experiment branch with at least one commit, click a history entry
    2. Confirm: compare toggle `[vs previous save] [vs main]` appears in the DiffViewer header
    3. Click `vs main` — confirm diff loads (showing differences from when the branch was created)
    4. Click `vs previous save` — confirm diff returns to the normal view
