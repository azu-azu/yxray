### Implementation Note: git_pushed_shas Fallback

The plan specified `git_pushed_shas` should use `@{u}` and return `set()` when no upstream is set. The implementation does this and also adds a graceful fallback: if `@{u}` is not set but `origin/<current_branch>` ref exists locally (from a previous push without `--set-upstream`), it uses that. This is a safe enhancement — the no-upstream test still passes because a fresh repo has no `origin` remote at all, causing the fallback log command to also return non-zero and return `set()`. The must-have truth ("returns empty set when no upstream is configured") is satisfied.
