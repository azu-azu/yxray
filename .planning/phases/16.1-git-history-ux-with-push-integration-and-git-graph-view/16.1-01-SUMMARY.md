## Decisions Made

- `git_pushed_shas` placed directly after `git_ahead_behind` in `git_ops.py` — both functions use the same `@{u}` upstream ref detection pattern; logical grouping aids readability
- `is_pushed` annotated in router (`list_history`) rather than in `git_log` service — keeps `git_ops` functions returning raw git data; router owns the API response shape
- Existing `test_list_history_endpoint` does not need `git_pushed_shas` mock — real subprocess call on `tmp_path` (no upstream configured) returns `set()` silently, assigning `is_pushed=False` to all entries; existing assertions on `sha` and `message` still pass without modification
