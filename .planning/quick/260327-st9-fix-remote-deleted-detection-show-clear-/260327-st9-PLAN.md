<objective>
Fix remote-deleted detection so users see a clear, actionable message instead of a cryptic subprocess error when their GitHub or GitLab repo no longer exists.

Purpose: When a repo is deleted on the remote, the push silently fails with `str(CalledProcessError)` — a full command dump that gives no guidance. Users have no idea their repo is gone and their URL is stale.

Output: Three-layer fix — git_ops detects the pattern and raises a typed exception, the router catches it and clears the stale URL, the frontend shows a clear message with a Retry button.
</objective>
