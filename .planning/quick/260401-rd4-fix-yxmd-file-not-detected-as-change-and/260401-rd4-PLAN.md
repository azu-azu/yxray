Key prior decisions:
- WORKFLOW_SUFFIXES frozenset at module level in git_ops.py — shared constant
- git_changed_workflows uses git status --porcelain
- GIT_ASKPASS pattern for git_push (never token in URL/args)

The user's test project at /Users/laxmikantmukkawar/test123 has:
- A file named `test.yxmd.rtf` (macOS TextEdit saved it with an extra .rtf extension)
- A git repo with NO commits yet (fresh init, nothing committed)

Root cause of Bug 1:
  Path("test.yxmd.rtf").suffix == ".rtf"  — not in WORKFLOW_SUFFIXES
  Path("test.yxmd.rtf").suffixes == [".yxmd", ".rtf"]  — .yxmd IS present
  Fix: use `any(s in WORKFLOW_SUFFIXES for s in Path(filename).suffixes)`

Root cause of Bug 2:
  git push on a repo with no commits exits non-zero with stderr like
  "error: src refspec main does not match any"
  This hits the generic CalledProcessError path → frontend shows "Check your connection"
  Fix: call git_has_commits() before push; return a NoPushableCommitsError so the
  router returns {"success": False, "error": "no_commits"} and the frontend can show
  a specific message like "Save your workflow first before pushing."
</context>
